# devops-satc
- O que é Devops e SRE

SRE (Site Reliability Engineering) e DevOps são práticas focadas em eficiência e automação, onde DevOps é uma cultura que une times de desenvolvimento e operações para acelerar entregas, enquanto SRE é uma implementação prática dessa cultura criada pelo Google, focada especificamente na confiabilidade, escalabilidade e performance de sistemas em produção.

DevOps (Cultura/Filosofia): Quebra barreiras entre equipes de desenvolvimento e operações, focando em automação (CI/CD), agilidade, colaboração e responsabilidade compartilhada para colocar software em produção mais rápido.
SRE (Engenharia/Prática): Aplica engenharia de software para resolver problemas operacionais. Um SRE foca em garantir que o sistema esteja disponível e estável, usando métricas como SLIs (Indicadores de Nível de Serviço) e SLOs (Objetivos de Nível de Serviço) e gerenciando "orçamentos de erro" (error budgets).
Relacionamento: SRE é considerado uma forma de implementar o DevOps. Enquanto DevOps foca no ciclo de vida do desenvolvimento, o SRE foca intensamente na produção/confiabilidade.
O que fazem: Ambos utilizam ferramentas de automação, monitoramento e IaC (Infraestrutura como Código), como Kubernetes, Prometheus, Terraform e Git.
